# ロードマップ３: 複数API対応

## 目標
単一APIモード（ローカルComfyUI or SDWebUI）から、複数のAPIプロバイダーを機能ごとに使い分けられる構造にする。

## 対応予定プロバイダー
- ローカルComfyUI（既存）
- ローカルSDWebUI/Forge（既存）
- RunPod ComfyUI（クラウド版ComfyUI）
- RunPod Endpoint（サーバーレスGPU）
- Fal.ai（クラウドAI API）

## フェーズ構成

| Phase | 内容 | 状態 |
|-------|------|------|
| Phase1 | プロバイダー抽象レイヤー導入 | 完了 |
| Phase2 | RunPod ComfyUI対応 | 完了 |
| Phase3 | RunPod Endpoint対応 | 未着手 |
| Phase4 | Fal.ai対応 | 未着手 |
| Phase5 | 機能別プロバイダー割り当てUI | 未着手 |

## Phase1で導入したファイル構成

```
js/ai/provider/
├── ai-provider.js          ... AIProvider基底クラス
├── provider-registry.js    ... プロバイダーレジストリ（登録・ルーティング）
├── local-comfyui-provider.js  ... ローカルComfyUIラッパー
└── local-sdwebui-provider.js  ... ローカルSDWebUI/Forgeラッパー
```

### 改修済みファイル
- `js/ai/ai-management.js` ... ルーターをproviderRegistry経由に変更
- `js/ai/ai-settings.js` ... プロバイダー登録・マッピング・初期同期
- `js/ai/ui/ai-ui-util.js` ... API切替時にレジストリ同期追加
- `index.html` ... scriptタグ追加

### Phase2で追加したファイル
- `js/ai/provider/runpod-comfyui-provider.js` ... RunPod ComfyUIプロバイダー

### Phase2で改修したファイル
- `js/ai/comfyui/comfyui-management.js` ... comfyuiFetch()/getComfyUIServerAddress()/getComfyUIAuthHeaders()追加、全fetch呼出をcomfyuiFetch()に置換、WebSocket認証対応
- `js/ai/comfyui/v2/comfyui-util-v2.js` ... fetch→comfyuiFetch置換、getComfyUIServerAddress()使用
- `js/ai/comfyui/v2/comfyui-workflow-editor.js` ... fetch→comfyuiFetch置換
- `js/ai/role/ai-roles.js` ... RUNPOD_COMFYUIロール追加
- `js/project-management.js` ... apis.RUNPOD_COMFYUI追加、SETTINGS_SCHEMA拡張
- `js/ai/ai-settings.js` ... RunPodプロバイダー登録、UIイベント
- `js/ai/ui/ai-ui-util.js` ... RunPodボタン分岐、isComfyUIMode()ヘルパー
- `index.html` ... RunPod設定UI、サイドバーボタン、scriptタグ

### apiModeとの互換性
- `apiMode`（グローバル変数）と`apis`定数はそのまま維持
- `hasRole()`/`hasNotRole()`（ai-roles.js）もそのまま維持
- `providerRegistry.syncFromApiMode(apiMode)`で`apiMode`→アクティブプロバイダーを同期
- 既存コードの`apiMode`参照箇所は変更不要
