接続ステータス表示の改善 — デザイン案

現状の表示

activeプロバイダー1つの状態しか表示されない。
"ComfyUI ON" がLocalなのかRunPodなのか区別できない。

activeがLocal ComfyUIの場合

ComfyUI ON

activeがRunPodの場合

RunPod ON

問題: T2I=Local、Inpaint=RunPod のように混在設定の場合、片方しか見えない。
"RunPod" だけでは ComfyUI なのか分かりにくい。

案A: 縦並びリスト表示

ロール割り当てで使用中のプロバイダーを全て縦に並べて表示。
ドット + プロバイダー名(種別) + ON/OFF で状態が一目瞭然。
未使用プロバイダーは非表示。

Local + RunPod 両方使用中

ComfyUI (Local) ON
ComfyUI (RunPod) OFF

Local のみ使用

ComfyUI (Local) ON

SD WebUI + Fal.ai（他プロバイダー混在）

SD WebUI ON
Fal.ai ON

全プロバイダー使用中（最大ケース）

ComfyUI (Local) ON
ComfyUI (RunPod) OFF
SD WebUI ON
Fal.ai ON

案B: チップ横並び表示

省スペース。チップ風のバッジで横に並べる。
ボーダー色で ON/OFF を表現。折り返し対応。

Local + RunPod 両方使用中

ComfyUI (Local) ComfyUI (RunPod)

Local のみ使用

ComfyUI (Local)

全プロバイダー使用中（折り返しケース）

ComfyUI (Local) ComfyUI (RunPod) SD WebUI Fal.ai

案の比較

現状 案A（縦並び） 案B（チップ横並び）
複数プロバイダー表示 1つだけ 全表示 全表示
Local/RunPod 区別 不可 明確 明確
省スペース 最小 中程度（縦に伸びる） コンパクト
視認性 低い 高い（一覧性） 中程度
実装コスト — 低い 低い

実装方針メモ

表示対象の判定: ロール割り当て (providerRegistry.getAllRoleAssignments()) で使用中のプロバイダーIDを収集。
割り当てが無い場合は activeProvider のみ表示。

heartbeat の変更: 現在は active プロバイダー1つだけ heartbeat する。
→ 使用中の全プロバイダーに対して個別に heartbeat を実行し、各ラベルを更新。

プロバイダー表示名の変更:

  • LocalComfyUIProvider.name: "ComfyUI" → "ComfyUI (Local)"
  • RunPodComfyUIProvider.name: "RunPod" → "ComfyUI (RunPod)"
  • SD WebUI, Fal.ai: 変更なし

HTMLの変更: ExternalService_Heartbeat_Label を単一 label → コンテナ div に変更。
JS側で使用中プロバイダーごとの status-item を動的生成。
